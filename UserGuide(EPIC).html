User Guide(API-T-DROID)

Generally, API-T-DROID(hereafter EPIC) is an easy to use API Test Tool. The program has been made user friendly, and one can start testing, with just a few steps.

This guide contains the following:

  1. How to setup EndPoints
  2. How to setup a Test Case
  3. How to setup a Test Epic
  4. How to create a Test Job
  5. How to run a Test Job

A. How to setup EndPoints

Endpoints are the urls where you will hit your test with, such as: https://192.168.0.162:5000/api/v1/goschool/getkey It is primarily comprised of 2 parts - the url and the action string.

Steps:

  1. Edit the Name.
  2. Edit the URL.
  3. Click OK, and Enter.

B. How to setup a Test Case

A test case is the basic, smallest unit for testing.
A test case is basically has these sections - action string, payload, and expected response.

Steps:

  1. Edit the Name, Description, Notes.
  2. Edit the Action string(eg. api/v1/goschool/getkey). Select the http Verb from the dropdown.
  3. Edit the request payload, which is typically json text.
  4. Edit the reponse required such as Status Code, Status Text, Body Text and Response Time.
  5. Click OK, and Submit.

C. How to setup a Test Epic

A test epic is a grouping of test cases, run in sequence.
This allows the user to conveniently test a scenario, which can be defined as an ordered set of actions.

Steps:

  1. Select the desired Test Case/Epic from the left box.
  2. Click the resepctive arrow.
  3. Click OK, and Create.

D. How to create a Test Job

A test job can be visualized as a compiled runnable grouping of unit.
To define a test job, pick an EndPoint, and the respective Test Case/Epic.

Steps:

  1. Select the desired EndPoint from the left box.
  2. Select the desired Test Case/Epic from the left box.
  3. Click the resepctive arrow.
  4. Click OK, and Create Job.

E. How to run a Test Job

A test job can be visualized as a compiled runnable grouping of unit.
To define a test job, pick an EndPoint, and the respective Test Case/Epic.

Steps:

  1. Select the desired Test Job from the table.
  2. Click OK, and Run.
